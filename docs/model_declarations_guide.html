1The DSL Goal

We want a Stan-like Python DSL where a model is just a decorated class:

@model
class LinearRegression:
    alpha = Param(Normal(0, 1))
    beta  = Param(Normal(0, 1))
    sigma = Param(Normal(0, 1), constraint=Positive())
    x     = Data()
    mu    = alpha + beta * x
    y     = Observed(Normal(mu, sigma))

And it must also support hierarchical priors:

n_groups = Data()
alpha = Param(Normal(alpha_pop, sigma_alpha), size=n_groups)
The trick: mu = alpha + beta * x runs immediately during class-body execution. So Param and Data must behave like symbolic expression objects, not just declarations.

2The Core Challenge

Inside a Python class body, statements execute top-to-bottom in real time. When the interpreter reaches:

mu = alpha + beta * x

…the attributes alpha, beta, x are still just local names with no idea what they’ll be called from the outside. The class hasn’t finished being built yet.

Problem: Declarations cannot know their final attribute names during class-body execution.

What we know inside the body

The objects themselves (the Param/Data instances).

What we don't know yet

The names Python will eventually bind them to.

3The Two-Step Approach

Instead of one fancy mechanism, the plan uses a clean two-step:

Step 1 · Class body runs
Each Param() / Data() tags itself with a unique UnresolvedSymbol(id). Expressions like alpha + beta * x build a tree of expression nodes that reference these symbols.
▼
Step 2 · @model decorator runs
The decorator walks cls.__dict__, learns each declaration's real attribute name, then rewrites every UnresolvedSymbol into the proper string like "alpha".
This is not a monad. It's a deferred symbolic expression graph with a single terminal operation: @model resolves symbols → names.

4UnresolvedSymbol Lifecycle

A tiny frozen dataclass placed in model/expr.py:

@dataclass(frozen=True)
class UnresolvedSymbol:
    id: int

type Symbol = str | UnresolvedSymbol

The lifecycle of a reference looks like this:

Inside class body
ParamRef(UnresolvedSymbol(0))
DataRef(UnresolvedSymbol(1))
▼ @model decorator
After decoration
ParamRef("alpha")
DataRef("x")
Why not just use a bare int for unresolved names? It works, but UnresolvedSymbol communicates intent and prevents accidental confusion with ordinary numeric constants like ConstNode(0).
Expression nodes (expr.py) must not know about Param / Data / @model. They only define the symbol type. Keep layers clean.

5Declarations vs References

Two concepts that look similar but stay separate:

Declarations

Param, Data, Observed — what you write in the class body.

Expression Refs

ParamRef, DataRef — nodes that appear inside expression trees.

To bridge the two, each declaration exposes .ref() and forwards arithmetic operators:

def __add__(self, other):
    return self.ref() + other

Operators implemented on declarations:

+ - * / reverse versions [ ]

Do not merge Param and ParamRef. One is a node in the metadata; the other is a node in expression trees. Mixing them collapses the two-layer design.

6The Two-Pass Decorator

The decorator runs in model/decorator.py and walks cls.__dict__ twice.

Pass 1 — collect declarations + symbol table

params: dict[str, Param]    = {}
data_slots: list[str]      = []
symbols: dict[UnresolvedSymbol, str] = {}

for name, value in cls.__dict__.items():
    if isinstance(value, Param):
        params[name] = value
        symbols[value.symbol] = name
    elif isinstance(value, Data):
        data_slots.append(name)
        symbols[value.symbol] = name

Pass 2 — observed + expression normalization

for name, value in cls.__dict__.items():
    if isinstance(value, Observed):
        observed_name = name
        observed = normalize_observed(value, symbols)
    elif is_expr_node(value):
        expressions[name] = normalize_expr(value, symbols)

Resulting metadata

@dataclass(frozen=True)
class ModelMeta:
    params: dict[str, Param]
    data_slots: list[str]
    observed_name: str
    observed: Observed
    expressions: dict[str, ExprNode]
Why two passes? Pass 1 has to build the complete symbol → name map before Pass 2 starts rewriting expressions; otherwise references would be normalized against an incomplete table.
Validation rules: exactly one Observed; no missing observed variable; no duplicates.

7Symbol Normalization

The decorator must rewrite every UnresolvedSymbol into the real string name across three places:

  1. Free-standing expressions (mu = alpha + beta * x)
  2. Distribution fields (Normal(mu, sigma) on the observed)
  3. Hierarchical prior fields (Normal(alpha_pop, sigma_alpha)) and size= on a Param

Before

BinOp(
  "+",
  ParamRef(UnresolvedSymbol(0)),
  BinOp("*",
    ParamRef(UnresolvedSymbol(1)),
    DataRef(UnresolvedSymbol(2))))

After

BinOp("+",
  ParamRef("alpha"),
  BinOp("*",
    ParamRef("beta"),
    DataRef("x")))

The recursive shape

def normalize_expr(expr, symbols):
    match expr:
        case ParamRef(name) if isinstance(name, UnresolvedSymbol):
            return ParamRef(symbols[name])
        case DataRef(name) if isinstance(name, UnresolvedSymbol):
            return DataRef(symbols[name])
        case ConstNode():
            return expr
        case BinOp(op, l, r):
            return BinOp(op, normalize_expr(l, symbols), normalize_expr(r, symbols))
        case IndexOp(base, idx):
            return IndexOp(normalize_expr(base, symbols), normalize_expr(idx, symbols))

Hierarchical priors need it too

A Param whose distribution contains other Params (like Normal(alpha_pop, sigma_alpha)) also gets normalized. So does the size= hint:

Before

Normal(loc=Param(...), scale=Param(...))
size=Data(...)

After

Normal(loc=ParamRef("alpha_pop"), scale=ParamRef("sigma_alpha"))
size=DataRef("n_groups")

8bind() and BoundModel

After decoration, the class gets a bind classmethod. Calling it pairs concrete data with the model and computes parameter shapes.

@dataclass(frozen=True)
class BoundModel:
    meta: ModelMeta
    data: dict[str, jax.Array]
    param_shapes: dict[str, tuple[int, ...]]
    n_params: int

What bind does

  1. Require all explicit data slots.
  2. Require the observed data (by observed_name).
  3. Reject extras.
  4. Convert everything to JAX arrays.
  5. Resolve parameter shapes from size=.
  6. Return BoundModel.

Shape resolution rules

if param.size is None:
    shape = ()
elif isinstance(param.size, int):
    shape = (param.size,)
elif isinstance(param.size, DataRef):
    shape = (int(bound_data[param.size.name]),)

Hierarchical example

bound = HierarchicalRegression.bind(
    n_groups=3,
    y=jnp.asarray([0.0, 1.0, 2.0]))

bound.param_shapes["alpha_pop"]   == ()
bound.param_shapes["sigma_alpha"] == ()
bound.param_shapes["alpha"]       == (3,)
bound.n_params == 5
data_slots contains only explicit Data() declarations; observed_name is kept separate but bind still requires the observed variable to be passed in.
BoundModel holds data + shapes only. No compiler, no sampler, no BlackJAX. That logic lives elsewhere.

9Design Invariants

Rules that must remain true across the whole feature:

Param and Data are declarations.
ParamRef and DataRef are symbolic references.
Class-body expressions use UnresolvedSymbol values.
The @model decorator normalizes UnresolvedSymbol values into real names.
Observed is not a Data slot, but bind(...) still requires observed data.
BoundModel contains data and shapes only; no inference logic.
No BlackJAX in model code.
No compiler logic in model code.
Prefer immutable dataclasses.
Do not use Any.
